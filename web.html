Security

HTTP vs HTTPS

The single "S" makes all the difference in security.

🌐

HTTP

  • ✕ Data sent in plain text
  • ✕ Vulnerable to interception
  • ✕ No identity verification
  • ✕ Flagged "Not Secure" by browsers
VS
🔒

HTTPS

  • ✓ Data encrypted with TLS/SSL
  • ✓ Protected from eavesdropping
  • ✓ Verified via SSL certificates
  • ✓ Shows the padlock icon

FAQ

Frequently Asked Questions

Quick answers about the internet's core protocols.

HTTPS is HTTP with an added encryption layer (TLS/SSL). It scrambles data between browser and server so attackers can't read or tamper with it, and it verifies the website's identity with a certificate.

FTP is optimized specifically for transferring large files or batches of files between computers, and gives more control over directory structures than HTTP file downloads typically offer.

POP3 downloads emails to a single device and often removes them from the server, while IMAP keeps emails on the server and syncs them across every device you use.

DNS translates human-friendly domain names like example.com into the numeric IP addresses computers actually use to locate each other on a network.

No — SMTP is only responsible for sending and relaying outgoing mail. Receiving and reading mail is handled by POP3 or IMAP instead.
